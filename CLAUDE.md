# CLAUDE.md — PlexHub Backend

## §1. Résumé
Backend **FastAPI** (Python, async) du stack PlexHub : miroir de bibliothèques **Xtream-IPTV** (VOD + séries + chaînes Live + EPG), enrichissement **TMDB**, validation de flux, génération de **bibliothèque Plex** (NFO + arborescence + .strm/images), **recommandations IA** (embeddings + recherche vectorielle) et **appairage TV** device-flow. Consommé par l'app Android `PlexHubTV`. Le backend ne stocke **aucun historique utilisateur côté IA** : `ai_tmdb_cache` + `ai_embeddings` sont clés sur `tmdb_id`. Cible de déploiement **Docker/Linux** : l'élection master-worker s'appuie sur `fcntl.flock` (POSIX, `app/main.py:196,226`).

- **§5.1 Sync** — `sync_worker.run_all_accounts()` → `sync_account(id)` (lock par compte) → `xtream_service` (catégories, VOD, séries+épisodes, Live, EPG) → upsert incrémental par `dto_hash`/`content_hash` + `differential_cleanup*`. - **§5.2 Enrichissement** — `enrichment_worker.run()` → `tmdb_service` (Phase 1 movies puis Phase 2 séries ; détails `append_to_response=credits,external_ids`), borné par `ENRICHMENT_DAILY_LIMIT` (défaut **50000**, compté en **appels API** réels). Concurrence 8, `MAX_ATTEMPTS=3`. Réf : `workers/enrichment_worker.py:151`, `config.py:37`.
- **§5.3 Validation de flux** — `health_check_worker.run_pipeline_validation()` (pipeline) et `run()` (cron) : HEAD puis Range GET (magic bytes), concurrence `STREAM_VALIDATION_CONCURRENCY` (défaut 20), marque cassé après `STREAM_BROKEN_THRESHOLD` (défaut 3) échecs ou échec **définitif** (404/403/error-CT/empty/magic-fail), re-check toutes `STREAM_VALIDATION_RECHECK_HOURS` (défaut 24), **circuit breaker** par compte à 90 % d'échecs. Réf : `workers/health_check_worker.py:274,89-103`.
- **§5.4 Dédup unifiée (DEUX consommateurs, logique partagée)** — la dédup par `unification_id` (imdb→tmdb→titre+année) vit dans **`aggregation_service`** et alimente **les deux** chemins :
  - **(a) API REST (app PlexHubTV)** — endpoints `GET /api/media/{movies,shows}/unified` + `/episodes/unified?unification_id=…` : `media_service.get_unified_list`/`get_unified_episodes` agrègent en mémoire les lignes `media` (catégories autorisées) → 1 `UnifiedMediaResponse` par titre avec `versions[]` (serverId/ratingKey/label/isBroken). Les **épisodes agrégés** sont regroupés par `(saison, épisode)` à travers tous les comptes du groupe (membres = shows de même `unification_id`, épisodes matchés par `(server_id, grandparent_rating_key)`) → 1 `UnifiedEpisodeResponse` par créneau avec ses `versions[]`. Les endpoints bruts par-ligne (`/movies`, `/shows`, `/episodes`) restent inchangés (non-cassant).
  - **(b) Génération de bibliothèque NFO/.strm (Plex + Jellyfin/Emby)** — `_auto_generate_plex_library()` (si `PLEX_LIBRARY_DIR`) : **un seul arbre plat dédupliqué sur TOUS les comptes actifs** (plus de `output/{account_id}`). `DatabaseSource()` réutilise `aggregation_service` → `PlexLibraryGenerator` → `LocalStorage`. Chaque titre = **1 dossier + 1 `movie.nfo`/`tvshow.nfo` + 1 poster/fanart** ; chaque source/qualité/langue = un `.strm` distinct nommé ` - Label` (films **et** épisodes — convention **Plex + Jellyfin**, plus de tag `{edition-}` propriétaire). Épisodes agrégés par `(saison, épisode)` dans `Season XX/`, 1 `.nfo` par fichier-version. Suppression **folder-aware** : le NFO partagé survit tant qu'≥1 version vit dans le dossier. Exposé via `POST /api/plex/generate` (`accountId` optionnel restreint l'agrégation) et CLI (`python -m app.cli generate --all|--account-id`).
  - **Aucune migration** : `unification_id` déjà peuplé au sync + recalculé à l'enrichissement. Réf : `services/aggregation_service.py`, `services/media_service.py`, `api/media.py`, `plex_generator/source.py`, `app/main.py:77-114`.

Réf : `main.py:273-306`.
- **§5.5 Recommandations IA** — `POST /api/ai/rank` (item→item) / `POST /api/ai/rank-multi` (centroïde pondéré décroissant 1.0,0.9,… min 0.1, `ai.py:323`) → résolution refs (imdb→tmdb via `find_by_imdb_id`) → `load_cached_vectors` → `hydrate_misses` (cap 20, timeout 10 s/tâche) → `cosine_rank`. `embedding_service` = **fastembed** `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` (384 dim, `embedding_service.py:26-27`) + **sqlite-vec** (`vec0 FLOAT[384]`), cache `ai_tmdb_cache`. ## §9. Pièges (house law — à respecter impérativement)
1. **Cold start IA ~30 s** au 1ᵉʳ `/rank` : fastembed télécharge + charge les poids ONNX (cold start documenté `embedding_service.py:11`). Les appels suivants sont rapides (singleton `_model`). Override via `AI_EMBED_MODEL` ; cache persistant via `AI_EMBED_CACHE_DIR`.
2. **3 motifs de 503 IA** (contractuels) : `AI service not configured` (`AI_API_KEY` vide, `deps.py:33-36`) · `AI vector storage unavailable` (sqlite-vec non chargé, `deps.py:37-41`) · `AI model unavailable` (fastembed KO → `EmbeddingUnavailableError`, `ai.py:216-219,306-311`).
3. **Épisodes/personnes non supportés par l'IA** : ranking au niveau `movie`/`tv` uniquement (`Literal["movie","tv"]`, `ai.py:72,88`). `find_by_imdb_id` ignore défensivement `tv_episode_results`/`tv_season_results`/`person_results` (`tmdb_service.py:204,225`) → comptés en `resolutionFailed` et droppés.
4. **Cap 20 fetches TMDB frais par `/rank`** : `HYDRATE_CAP = 20`, timeout 10 s/tâche (`recommendation_service.py:28-29,169-170`) ; le surplus + timeouts + échecs → `cacheMissesDropped` (`stats.dropped`). Le client re-appelle une fois le cache chaud.
5. **Le rebuild ne tourne JAMAIS au boot** : uniquement via `POST /api/ai/embed/rebuild` (`embedding_worker.py:3,53`). Idempotent : scan `embedded_at IS NULL`, curseur `tmdb_id > :cursor` (`embedding_worker.py:76`), `PAGE_SIZE=50`, DELETE-puis-INSERT sur `vec0` (UPSERT interdit sur table virtuelle, `embedding_worker.py:108-116`). Jobs en mémoire, `JOBS_CAP=100`.
6. **Migrations idempotentes** : tout DDL en `IF NOT EXISTS` / `ADD COLUMN` gardé ; nouvelle migration **en fin** de `run_migrations()`. **Ordre d'exécution : 001-007, puis 008 (sur connexion), puis 009** (`migrations.py:20-31`). M008 (`ai_embeddings` vec0 `FLOAT[384]` + `ai_tmdb_cache`, `migrations.py:272-306`) **dépend du chargement de sqlite-vec** (`register_sqlite_vec_listener`, `database.py:25-54`). Migration courante = **009** (`tv_auth_sessions`).
7. **`fcntl.flock` = POSIX** : l'élection master-worker (`main.py:196,226`) ne fonctionne que sous Linux/Docker (pas Windows natif). Seul le master lance le scheduler ; le lock libéré releases le flock (`main.py:333-334`).
8. **SQLite « database is locked »** : passer par `commit_with_retry` / `run_with_retry` (`db_retry.py`, délais 0.1/0.5/1.0 s au-dessus du `busy_timeout=5000`). WAL activé. Ne jamais ouvrir plusieurs writers sans retry.
9. **Rotation de logs Windows** : `SafeRotatingFileHandler` avale les `PermissionError` (`main.py:30-35`) quand un autre process tient le fichier. Ne pas supprimer ce garde-fou.
10. **CORS** : `CORS_ORIGINS` **défaut `*`** (`config.py:55-57`) — **expliciter en prod**. Tokens/clés jamais loggés ni renvoyés. ⚠️ Rappel : seuls `/api/ai` et `/api/tv-auth/approve` sont authentifiés — durcir l'exposition publique des autres routers.
11. **Appels bloquants** (`sqlite3.backup` `main.py:296`, init/inférence ONNX `embedding_service.py:78,124`) → `asyncio.to_thread` ; ne jamais bloquer la boucle d'événements. Les téléchargements d'images Plex passent par un `ThreadPoolExecutor` dédié (`storage.py:36`).
