# CLAUDE.md — PlexHub Backend

## §1. Résumé
Backend **FastAPI** (Python, async) du stack PlexHub : miroir de bibliothèques **Xtream-IPTV** (VOD + séries + chaînes Live + EPG), enrichissement **TMDB**, validation de flux, génération de **bibliothèque Plex** (NFO + arborescence + .strm/images), **recommandations IA** (embeddings + recherche vectorielle), **LLM Ollama/gemma4** (descriptions, chat, **traduction de sous-titres FR**) et **appairage TV** device-flow. Consommé par l'app Android `PlexHubTV`. Le backend ne stocke **aucun historique utilisateur côté IA** : `ai_tmdb_cache` + `ai_embeddings` sont clés sur `tmdb_id`. Cible de déploiement **Docker/Linux** : l'élection master-worker s'appuie sur `fcntl.flock` (POSIX, `app/main.py:196,226`).

## §9. Pièges (house law — à respecter impérativement)
1. **Cold start IA ~30 s** au 1ᵉʳ `/rank` : fastembed télécharge + charge les poids ONNX (cold start documenté `embedding_service.py:11`). Les appels suivants sont rapides (singleton `_model`). Override via `AI_EMBED_MODEL` ; cache persistant via `AI_EMBED_CACHE_DIR`.
2. **4 motifs de 503 IA** (contractuels) : `AI service not configured` (`AI_API_KEY` vide, `deps.py:33-36`) · `AI vector storage unavailable` (sqlite-vec non chargé, `deps.py:37-41`) · `AI model unavailable` (fastembed KO → `EmbeddingUnavailableError`, `ai.py`) · **`LLM unavailable — Ollama unreachable`** (gemma4 injoignable → `_ollama_503`, `ai.py:499` ; couvre `/describe`·`/chat`·`/subtitles/translate`).
3. **Épisodes/personnes non supportés par l'IA (ranking)** : ranking au niveau `movie`/`tv` uniquement (`Literal["movie","tv"]`). `find_by_imdb_id` ignore défensivement `tv_episode_results`/`tv_season_results`/`person_results` → comptés en `resolutionFailed` et droppés. (La traduction de sous-titres §5.7 est indépendante de cette limite.)
4. **Cap 20 fetches TMDB frais par `/rank`** : `HYDRATE_CAP = 20`, timeout 10 s/tâche (`recommendation_service.py:28-29`) ; le surplus + timeouts + échecs → `cacheMissesDropped` (`stats.dropped`). Le client re-appelle une fois le cache chaud.
5. **Le rebuild ne tourne JAMAIS au boot** : uniquement via `POST /api/ai/embed/rebuild` (`embedding_worker.py:3,53`). Idempotent : scan `embedded_at IS NULL`, curseur `tmdb_id > :cursor`, `PAGE_SIZE=50`, DELETE-puis-INSERT sur `vec0` (UPSERT interdit sur table virtuelle). Jobs en mémoire, `JOBS_CAP=100`.
6. **Migrations idempotentes** : tout DDL en `IF NOT EXISTS` / `ADD COLUMN` gardé ; nouvelle migration **en fin** de `run_migrations()`. **Ordre d'exécution : 001-007, puis 008 (sur connexion), puis 009, 010, puis 011**. M008 (`ai_embeddings` vec0 `FLOAT[384]` + `ai_tmdb_cache`) **dépend du chargement de sqlite-vec** (`register_sqlite_vec_listener`, `database.py:25-54`). Migration courante = **011** (`ai_media_blurb`, table additive idempotente). M010 = `ai_subtitle_cache`.
7. **`fcntl.flock` = POSIX** : l'élection master-worker (`main.py:196,226`) ne fonctionne que sous Linux/Docker (pas Windows natif → `uvicorn app.main:app` échoue au boot sous Windows). Seul le master lance le scheduler ; le lock libéré releases le flock.
8. **SQLite « database is locked »** : passer par `commit_with_retry` / `run_with_retry` (`db_retry.py`, délais 0.1/0.5/1.0 s au-dessus du `busy_timeout=5000`). WAL activé. Ne jamais ouvrir plusieurs writers sans retry.
9. **Rotation de logs Windows** : `SafeRotatingFileHandler` avale les `PermissionError` (`main.py:30-35`) quand un autre process tient le fichier. Ne pas supprimer ce garde-fou.
10. **CORS** : `CORS_ORIGINS` **défaut `*`** (`config.py:55-57`) — **expliciter en prod**. Tokens/clés jamais loggés ni renvoyés. ⚠️ Rappel : seuls `/api/ai` et `/api/tv-auth/approve` sont authentifiés — durcir l'exposition publique des autres routers.
11. **Appels bloquants** (`sqlite3.backup`, init/inférence ONNX `embedding_service.py:78,124`) → `asyncio.to_thread` ; ne jamais bloquer la boucle d'événements. Les téléchargements d'images Plex passent par un `ThreadPoolExecutor` dédié (`storage.py:36`). Les appels LLM Ollama passent par httpx async (timeouts dédiés `ollama_service.py:21-22`).
12. **Amplification LLM sous-titres** : `POST /api/ai/subtitles/translate` peut déclencher de nombreux appels Ollama (1 par lot de `SUBTITLE_CHUNK_CUES` cues). Bornes : `SUBTITLE_MAX_CUES`/`SUBTITLE_MAX_BYTES` (rejet 413), concurrence `SUBTITLE_CONCURRENCY`, deadline `SUBTITLE_TOTAL_TIMEOUT`. Le cache `ai_subtitle_cache` amortit ; ne pas relâcher ces bornes sans rate-limit.
