## backend-developer — C6 (UI admin « Télécharger Plex »)

Nouvel onglet admin `/admin/plex-downloads` livré sur `develop`, miroir strict de l'onglet
Xtream `/admin/downloads`, mais lisant le catalogue Plex dédupliqué (`plex_catalog_service`,
tables `plex_server`/`plex_media_item`) et enfilant via `plex_download_service.enqueue_plex_selection`.
**Créés** : `app/api/admin_plex_downloads.py` (router HTMX, 0 logique métier — délègue à
`plex_catalog_service`/`plex_download_service`/`plex_sync_service`), 5 templates
(`plex_downloads.html`, `_plex_downloads_list.html`, `_plex_downloads_versions.html`,
`_plex_downloads_episodes.html`, `_plex_sync_status.html`), `tests/test_admin_plex_downloads.py`
(30 tests). **Modifiés (minimal)** : `app/services/plex_catalog_service.py` (+1 fonction
`get_sync_status` lisant le singleton `plex_sync_status`), `app/templates/admin/base.html`
(+1 lien nav), `app/main.py` (mount du router derrière le même `verify_admin_basic_auth` que
`admin_downloads.router`, + reap best-effort `plex_sync_service.reap_sync_status` au boot master
via une coroutine `_reap_plex_sync` dédiée, gardée `if settings.PLEX_ACCOUNT_TOKEN:`, dispatchée
non-bloquante via `create_background_task` — indépendante de `DOWNLOAD_DIR`/`_run_download_worker`).

Le POST d'enqueue unique (`POST /admin/plex-downloads`) route sur 4 scopes : `movie`/`series_all`
parsent un champ `source="server_id|rating_key"` (1er `|` seulement) ; `seasons` parse une liste
répétée `season_pick="season|server_id|show_rating_key"` (entrées vides = placeholder « ne pas
télécharger », ignorées) ; `episodes` idem sur `episode_pick="server_id|episode_rating_key"`. Le
fragment réponse (`_downloads_queue.html`, PARTAGÉ avec l'onglet Xtream — même table
`download_job`) est re-rendu via un `_queue_context`/`_job_view` répliqués à l'identique de
`admin_downloads.py` (duplication volontaire, périmètre de fichiers disjoint). `POST /sync`
est un no-op sûr si `PLEX_ACCOUNT_TOKEN` est vide (bandeau « configurer PLEX_ACCOUNT_TOKEN »,
aucune tâche lancée) ; sinon dispatch `plex_sync_service.run_full_sync` en tâche de fond
(import paresseux d'`async_session_factory` DANS le handler, pas au niveau module, pour rester
monkeypatchable en test — même raison que le pattern déjà utilisé par `main.py`). Le fragment
`_plex_sync_status.html` s'auto-poll (`hx-trigger="every 2s"`) tant que `state=='running'` puis
s'arrête de lui-même. Taille formatée côté routeur (`_fmt_size`, Gio si ≥1 Gio sinon Mio) —
jamais d'arithmétique dans les templates.

**Décision de design** : `_feature_enabled()` est une fonction (pas une constante figée à
l'import) pour rester sensible à `monkeypatch.setattr(settings, "PLEX_ACCOUNT_TOKEN", ...)` en
test — une capture au chargement du module aurait figé la valeur avant que les tests ne la
changent.

Invariant sécurité vérifié empiriquement (5 tests dédiés `TestNoSecretLeak`) : aucune page
(index/liste/versions/épisodes/statut sync/file) ne contient jamais `PlexServer.access_token`
ni `base_uri`, même quand des serveurs avec ces champs peuplés sont en base — `list_servers`/
`get_group`/`list_seasons_with_sources`/`list_episodes_with_sources` ne les exposent jamais côté
service, et ce routeur ne les lit nulle part.
